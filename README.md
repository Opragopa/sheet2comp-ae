# Sheets to AE Comp Generator

Набор скриптов для After Effects, которые создают композиции из таблиц, подставляют текст и фото, готовят данные из Google Sheets/TSV/CSV и помогают привести имена файлов в Render Queue к нужному формату.

## Что Запускать

| Задача | Скрипт | Где запускать |
| --- | --- | --- |
| Простые плашки из Google Sheets: имя + регалии в текстовых слоях | `Sheets-to-AE-Comp-Generator.jsx` | After Effects |
| Плашки/визитки персон с фото из Google Sheets или локальной таблицы | `person_plates_from_sheet.jsx` | After Effects |
| Добавить выбранные композиции в Render Queue и сократить имена файлов | `shorten_render_queue_names.jsx` | After Effects |
| Подготовить общий контент-план и открыть генераторы тем/плашек/визиток | `content_plan_from_sheet.jsx` | After Effects |
| Композиции с темами сессий из TSV/CSV/Google Sheet | `session_topics_from_sheet.jsx` | After Effects |
| Разобрать общий контент-план по дням на темы, плашки и визитки | `extract_content_plan.py` | Терминал / GUI |
| Фоново следить за обновлениями двух Google Sheets | `sheet_update_monitor.py` | macOS / Терминал |
| Заранее скачать и переименовать фото для персон | `prepare_person_plate_photos.py` | Терминал |
| Достать темы сессий из большой программной таблицы в TSV | `extract_session_topics.py` | Терминал |
| Найти файлы на Яндекс.Диске по именам и выгрузить ссылки | `yandex_disk_links_from_names.py` | Терминал |
| Общая GUI-панель для Python-утилит | `tools_gui.py` | Python GUI |
| Запустить GUI-панель двойным кликом | `run_tools_gui_macos.command` / `run_tools_gui_windows.bat` | macOS / Windows |
| Скачать CSV и сохранить `data.json` для старого генератора | `download_data.py` | Вызывает `Sheets-to-AE-Comp-Generator.jsx` |
| Скачать CSV, фото и сохранить JSON для фото-плашек | `download_person_plate_data.py` | Вызывает `person_plates_from_sheet.jsx` и `prepare_person_plate_photos.py` |

## Требования

- Adobe After Effects 2024+
- Python 3
- Python-пакеты из `requirements.txt`
- Для Google Sheets: таблица должна быть доступна по ссылке или опубликована/export-доступна в CSV/TSV

Кириллица поддерживается: скрипты читают и пишут UTF-8, JSON сохраняется с русскими символами, а имена файлов чистятся только от запрещенных символов `\ / : * ? " < > |`.

## Простые Плашки

Используйте `Sheets-to-AE-Comp-Generator.jsx`, если нужно быстро создать композиции из таблицы, где имя и регалии берутся из колонок и записываются прямо в текстовые слои.

### Таблица

Минимальные колонки:

```csv
ФИО спикера,Должность
Иванов Иван Иванович,Директор
```

ФИО автоматически приводится к виду `ИМЯ ФАМИЛИЯ`: отчество отбрасывается, переносы строк чистятся, варианты `Фамилия Имя Отчество`, `Фамилия Имя` и `Имя Фамилия` приводятся к одному формату.

### Ручной список

Если таблицы нет, в окне генератора выберите `Источник: Вручную` и вставьте список:

```text
Иванов Иван Иванович | Директор
Петрова Анна; Руководитель проекта
```

Одна строка создает одну плашку. Первое поле считается ФИО, второе - должностью/регалиями. Разделители: `|`, `;` или табуляция. Первая строка с заголовками `ФИО | Должность` допускается и будет пропущена.

Примеры:

```text
Элла Памфилова -> ЭЛЛА ПАМФИЛОВА
Федоренко Константин Альбертович -> КОНСТАНТИН ФЕДОРЕНКО
Любимова Ольга -> ОЛЬГА ЛЮБИМОВА
Васильев Владимир Абдуалиевич -> ВЛАДИМИР ВАСИЛЬЕВ
```

### Запуск

1. Откройте проект After Effects.
2. Выделите композицию-шаблон в Project.
3. Запустите `File > Scripts > Sheets-to-AE-Comp-Generator.jsx`.
4. Выберите источник: `Таблица` для Google Sheets/CSV export или `Вручную` для списка строк.
5. Укажите колонки имени и должности, обычно `ФИО спикера` и `Должность`.
6. Укажите текстовые слои: обычно `ИМЯ` и `РЕГАЛИИ`.
7. Проверьте префикс и разделитель, обычно `Плашка` и `_`.
8. Нажмите `Проверить`, затем `Создать`.

### Результат

Композиции называются так:

```text
Плашка_ИМЯ ФАМИЛИЯ
```

Скрипт записывает `ИМЯ ФАМИЛИЯ` в слой имени обычным Source Text. Для слоя регалий он ставит Source Text expression автоподгонки размера: текст берется из колонки `Должность`, а размер управляется слайдерами `CONTROL`: `Regalia Base Size`, `Regalia Min Size`, `Regalia Chars Per Line`, `Regalia Max Lines`, `Regalia Manual Size`. Старый expression, который читал `thisComp.name.split(...)`, заменяется.

## Плашки С Фото

Используйте `person_plates_from_sheet.jsx`, если нужно создать плашки или визитки персон с именем, должностью и фотографией из выбранной таблицы.

### Что Делает

- берет данные из выбранной Google-таблицы/листа или локального CSV/TSV/TXT;
- скачивает фотографии из колонки `Фото на плашку`;
- поддерживает прямые ссылки на изображения, Google Drive file links и публичные ссылки/папки Яндекс.Диска;
- ищет уже скачанные фото в выбранной папке и не скачивает их повторно;
- сравнивает имена файлов в выбранной папке с `ИМЯ ФАМИЛИЯ` из таблицы, переименовывает найденные фото в единый формат и использует их в After Effects;
- для папок Яндекс.Диска выбирает файл по совпадению имени и фамилии в названии файла;
- фильтрует строки по колонке `Смена`, например только `единство`;
- дублирует активную композицию-шаблон;
- записывает `ИМЯ ФАМИЛИЯ` в текстовый слой `ИМЯ`;
- записывает должность в текстовый слой `ДОЛЖНОСТЬ`;
- заменяет фото в слое `Rectangle 3` или fallback-слое N6;
- умеет работать с фото-слоем внутри прекомпозиции;
- для визиток дублирует фото-прекомп и называет его `PHOTO_ИМЯ ФАМИЛИЯ`;
- опционально добавляет созданные композиции в Render Queue.

### Таблица

Ожидаемые колонки. Для фото можно использовать `Ссылка на плашку`, `Фото на плашку`, `ФОТО` или `Фото`. Если в одной фото-колонке текстовая пометка, а в другой ссылка, скрипт выберет ссылку:

```csv
ФИО,Должность,Ссылка на плашку,Смена
Иванов Иван Иванович,Директор,https://...,единство
```

### Запуск

1. Откройте проект After Effects.
2. Подготовьте композицию-шаблон с текстовыми слоями `ИМЯ`, `ДОЛЖНОСТЬ` и фото-плейсхолдером.
3. Запустите `File > Scripts > person_plates_from_sheet.jsx`.
4. Во вкладке `Данные` вставьте ссылку на таблицу или конкретный лист.
5. Если нужно переключить вкладку, укажите `GID листа` или нажмите `Взять из ссылки`, когда ссылка уже открыта на нужном листе.
6. В поле `Папка фото` выберите папку для скачивания и поиска фото.
7. Проверьте колонки `ФИО спикера`, `Должность`, `Фото на плашку`, `Смена`.
8. Если фото хотите поставить вручную, снимите галочку `Подтягивать фото автоматически`.
9. В поле `Выгрузить смены` укажите нужную смену. Если `Колонка смены` и `Выгрузить смены` пустые, будут выгружены все строки.
10. Во вкладке `Создание` выберите `Композиция-шаблон`. Кнопка `Активная` подставляет текущую активную композицию из Project/Timeline.
11. Во вкладке `Слои` проверьте имена слоев или fallback-номера.
12. Нажмите `Проверить`, затем `Создать`.

Если часть композиций уже была создана старой версией скрипта или данные в таблице поменялись, нажмите `Обновить`. Скрипт сначала покажет план: какие композиции будут изменены, какие останутся без изменений, где есть конфликты и что именно поменялось в слоях `ИМЯ`/`ДОЛЖНОСТЬ`/фото. Изменения применяются только после подтверждения.

Новые и найденные плашки получают служебный ключ в `Comment` композиции. Благодаря этому повторное обновление находит композицию даже после ручного переименования в Project, например если `10-30_Аширов Д.В.` переименовали в `10-30_Аширов Денис`.

### Имена И Слои

Композиции называются так:

```text
Визитка_Фамилия Имя
```

Для обычных таблиц имя композиции строится в порядке `Фамилия Имя` без номера и должности. Если TSV пришел из контент-плана и в нем есть время выступления, плашки называются по времени и человеку, например `09-30_Таньшина Наталья` и `11-00_Таньшина Наталья`. Сокращения и полные имена одного человека склеиваются по фамилии и первой букве имени: `Морская Е.` и `Морская Елизавета` станут `Морская Елизавета`. Для повторных выступлений первая по времени плашка создается полноценно, а следующие композиции этого спикера содержат внутри только первую плашку, чтобы правки в базовой композиции подтягивались дальше. Должность записывается прямо в текстовый слой `ДОЛЖНОСТЬ`, а слой `ИМЯ` остается в формате `ИМЯ ФАМИЛИЯ`.

Для слоя `ДОЛЖНОСТЬ` скрипт ставит Source Text expression автоподгонки размера. Expression читает текущий текст самого слоя, поэтому должность автоматически подтягивается из таблицы при создании, но ее можно потом вручную изменить в After Effects через `Source Text`. Слайдеры размера ищутся сначала на слое `CONTROL`, затем на `CONTROLLER`: `Regalia Base Size`, `Regalia Min Size`, `Regalia Chars Per Line`, `Regalia Max Lines`, `Regalia Manual Size`.

По умолчанию используются:

- имя: слой `ИМЯ`, fallback N3;
- должность: слой `ДОЛЖНОСТЬ`, fallback N4;
- фото: слой `Rectangle 3`, fallback N6.

Выбранная `Композиция-шаблон` сохраняется в `~/Documents/ae_person_plate_settings.json`, поэтому после перезапуска скрипта не нужно выбирать ее заново. Если шаблон не выбран, первый запуск подставит активную композицию, если она есть.

Если в колонке фото написан обычный текст без URL, композиция создается без фото и без ошибки загрузки.

### Ручное Добавление Людей

Если человека нет в таблице, во вкладке `Данные` выберите `Источник данных: Вручную`, затем откройте вкладку `Вручную`.

Одна строка - один человек:

```text
Имя Фамилия | Должность | Фото/ссылка | Смена
```

Можно указать только имя:

```text
Иван Иванов
Мария Петрова | Директор программы
Павел Сидоров | Продюсер | /Users/opragopa/Desktop/photo.jpg
Анна Смирнова | Куратор | https://disk.yandex.ru/...
```

Разделители: `|`, табуляция или `;`. Первая строка с заголовками `ФИО | Должность | Фото | Смена` допускается и будет пропущена. Ручной список сохраняется во временный TSV и проходит через тот же подготовщик, что и таблица: фото ищутся в выбранной папке, ссылки скачиваются, имена композиций и опция пересоздания работают одинаково.

В ручном режиме строки без смены не отбрасываются текущим фильтром `Выгрузить смены`. Если в четвертом поле указать смену, фильтр применится как обычно.

Порядок поиска фото:

1. Сначала скрипт ищет в выбранной папке файл с уже правильным именем `ИМЯ ФАМИЛИЯ.ext`.
2. Потом рекурсивно сравнивает остальные изображения в папке и вложенных папках с именем и фамилией из таблицы, включая порядок `ФАМИЛИЯ ИМЯ`, и переименовывает совпавший файл в `ИМЯ ФАМИЛИЯ.ext`.
3. Если локально совпадения нет, скачивает фото по ссылке из колонки `Фото на плашку`.
4. Если ссылка ведет на папку Яндекс.Диска, берет только файл, в названии которого совпали имя и фамилия. Если совпадения нет, строка получает ошибку фото, чтобы не подставить чужой портрет.

### Визитка С Фото-Прекомпом

Для визитки используйте композицию-шаблон, где внешний слой фото называется `PHOTO` или `PHOTO_TEMPLATE` и является прекомпом. Track Matte, luma-переход, рамку и маски оставляйте в шаблоне как обычно: скрипт не трогает эти связи в основной композиции.

Во вкладке `Создание` выберите тип `Визитка`. Скрипт включит строгий режим: слой `PHOTO` должен быть прекомпом. Для каждой строки он:

- дублирует основную композицию;
- называет новую композицию `Визитка_Фамилия Имя`;
- дублирует source-прекомп слоя `PHOTO`;
- называет копию `PHOTO_ИМЯ ФАМИЛИЯ`;
- заменяет фото внутри этой копии;
- подставляет персональный `PHOTO_...` обратно в созданную визитку.

Если композиция `Визитка_Фамилия Имя` уже есть в той же папке проекта, настройка `Пересоздать уже созданные композиции` во вкладке `Создание` решает поведение: выключена - существующая композиция остается как есть и строка пропускается; включена - старая композиция удаляется и создается новая.

Внутри прекомпа `PHOTO` ничего указывать не нужно. Скрипт сам возьмет первый файловый фото/footage-слой внутри `PHOTO`, заменит его на нужный портрет и подгонит по размеру. Если внутри `PHOTO` пока нет фото-слоя, скрипт добавит фото новым слоем.

Если выключить `Подтягивать фото автоматически`, скрипт не скачивает фото и не трогает содержимое `PHOTO`. Для визиток он все равно дублирует фото-прекомп в отдельный `PHOTO_ИМЯ ФАМИЛИЯ`, чтобы каждую визитку можно было поправить вручную независимо от остальных.

Генерация работает в режиме best effort: если для одной строки не найдено фото, композиция все равно создается без фото; если не найден один текстовый слой, остальные найденные слои заполняются. Все такие проблемы собираются в итоговый отчет.

## Подготовка Фото Без AE

Используйте `prepare_person_plate_photos.py`, если нужно заранее скачать фото из справочника до открытия After Effects.

```sh
python3 prepare_person_plate_photos.py --photos-dir "/Users/opragopa/Desktop/person-photos"
```

Для другой таблицы или вкладки:

```sh
python3 prepare_person_plate_photos.py --sheet-url "https://docs.google.com/spreadsheets/d/.../edit#gid=123456" --sheet-gid 123456 --photos-dir "/Users/opragopa/Desktop/person-photos"
```

По умолчанию скрипт сохраняет:

```text
~/Documents/ae_plaque_data/person_plate_photos
~/Documents/ae_plaque_data/person_plates_data.json
```

Файлы фото называются по формату:

```text
ИМЯ ФАМИЛИЯ.jpg
```

## Ссылки С Яндекс.Диска По Именам

Используйте `yandex_disk_links_from_names.py`, если нужно без Apps Script вставить в таблицу список имен и получить рядом ссылки на найденные файлы Яндекс.Диска.

Подготовьте текстовый файл, где в каждой строке одно имя:

```text
Иван Иванов
Мария Петрова
```

Задайте OAuth-токен Яндекс.Диска:

```sh
export YANDEX_DISK_TOKEN="..."
```

Сгенерируйте TSV:

```sh
python3 yandex_disk_links_from_names.py names.txt --root "disk:/Фото" --publish -o links.tsv
```

Результат можно открыть или вставить в Google Sheets/Excel. Колонки:

```text
Имя	Ссылка	Файл	Путь	Статус
```

Если входной файл уже CSV/TSV с заголовком, укажите колонку:

```sh
python3 yandex_disk_links_from_names.py people.tsv --column "ФИО спикера" --root "disk:/Фото" --publish -o links.tsv
```

Флаг `--publish` делает найденный файл доступным по публичной ссылке. Без него скрипт вернет только уже опубликованные ссылки и пометит остальные строки статусом `найдено, но нет публичной ссылки; добавь --publish`.

## Общая GUI-Панель

Используйте `tools_gui.py`, если не хочется запускать Python-утилиты через Терминал.

```sh
python3 tools_gui.py
```

На macOS можно запускать двойным кликом:

```sh
chmod +x run_tools_gui_macos.command
./run_tools_gui_macos.command
```

На Windows:

```bat
run_tools_gui_windows.bat
```

Вкладки GUI:

- `Фото плашек` запускает `prepare_person_plate_photos.py`;
- `Темы сессий` запускает `extract_session_topics.py`;
- `Яндекс.Диск` запускает `yandex_disk_links_from_names.py`;
- `Служебные` запускает `download_data.py` и `download_person_plate_data.py`.

GUI использует стандартный `tkinter`, поэтому отдельные Python-пакеты для окна не нужны.

## Добавление В Render Queue

Используйте `shorten_render_queue_names.jsx`, если нужно добавить выбранные композиции в Render Queue без дублирования композиций в Project.

Скрипт подходит и для старых композиций с именами:

```text
Плашка_Имя Фамилия_Должность
```

Файлы при рендере будут называться без должности:

```text
Плашка_Имя Фамилия.mov
```

Папка экспорта создается рядом с сохраненным `.aep` проектом:

```text
<папка проекта>/EXPORT
```

### Запуск

1. Сохраните проект After Effects.
2. В Project выделите одну или несколько композиций.
3. Запустите `File > Scripts > shorten_render_queue_names.jsx`.
4. Оставьте разделитель `_`, если имена собраны через подчеркивание.
5. Скрипт добавит выбранные композиции в Render Queue и выставит Output Module в папку `EXPORT`.
6. Проверьте примеры в финальном сообщении скрипта.

Скрипт не дублирует и не переименовывает композиции. Он добавляет в очередь именно выбранные `CompItem`.

Кириллица сохраняется. Скрипт не транслитерирует русские буквы и чистит только запрещенные для файлов символы.

## Контент-План По Дням

Используйте `extract_content_plan.py`, если в одной программной Google Sheet лежат сразу темы сессий, плашки и визитки по дням.

В After Effects удобнее запускать `File > Scripts > content_plan_from_sheet.jsx`. В нем есть:

- кнопка `Подготовить TSV`, которая запускает `extract_content_plan.py`;
- кнопка `Проверить изменения`, которая сравнивает текущую таблицу с последним `import_report.json` и предлагает обновить TSV, если данные изменились;
- кнопка `Темы сессий`, которая открывает `session_topics_from_sheet.jsx` на `content_plan_sessions.tsv`;
- кнопка `Плашки`, которая открывает `person_plates_from_sheet.jsx` на `content_plan_plates.tsv`;
- кнопка `Визитки`, которая открывает `person_plates_from_sheet.jsx` на `content_plan_cards.tsv` и включает тип `Визитка`.

Окно сохраняет последнюю ссылку, папку TSV, выбранный день и статус в:

```text
~/Documents/ae_content_plan_settings.json
```

По умолчанию используется папка `~/Documents/ae_plaque_data/content_plan`, чтобы не зависеть от кириллицы в путях проекта After Effects. Если выбранная папка не существует, скрипт спросит подтверждение перед созданием. Если парсер нашел `0` строк, TSV не записываются и показывается диагностическая ошибка. Диагностика передается в After Effects через отдельный UTF-8 JSON, поэтому русские ошибки не должны превращаться в нечитаемый текст. Последний отчет `import_report.json` содержит `data_hash`, по которому окно понимает, изменилась ли таблица после прошлой подготовки.

Если удобнее готовить данные без AE, используйте тот же парсер из терминала:

```sh
python3 extract_content_plan.py "https://docs.google.com/spreadsheets/d/10C3eoaG146WgOeQeoli90dQCHPruoJ_d4_rqcyoUR8M/edit?gid=213088400#gid=213088400"
```

По умолчанию результаты сохраняются в:

```text
~/Documents/ae_plaque_data/content_plan
```

Скрипт создает:

```text
content_plan_venues.tsv
content_plan_topics.tsv
content_plan_sessions_model.tsv
content_plan_session_people.tsv
content_plan_people.tsv
content_plan_badges.tsv
content_plan_cards_model.tsv
content_plan_sessions.tsv
content_plan_plates.tsv
content_plan_cards.tsv
content_plan_all_people.tsv
import_report.json
```

Новая модель лежит в `*_model.tsv`, `content_plan_people.tsv`, `content_plan_session_people.tsv`, `content_plan_badges.tsv`, `content_plan_cards_model.tsv` и `import_report.json`.

Совместимые файлы для текущих AE-генераторов:

- `content_plan_sessions.tsv` открывайте через `session_topics_from_sheet.jsx`;
- `content_plan_plates.tsv` открывайте через `person_plates_from_sheet.jsx` как `Плашка`;
- `content_plan_cards.tsv` открывайте через `person_plates_from_sheet.jsx` как `Визитка`;
- `content_plan_all_people.tsv` теперь дедуплицированный справочник людей, а не список всех появлений.

Чтобы выгрузить только один день:

```sh
python3 extract_content_plan.py "https://docs.google.com/spreadsheets/d/.../edit?gid=213088400#gid=213088400" --day "ДЕНЬ 3"
python3 extract_content_plan.py "https://docs.google.com/spreadsheets/d/.../edit?gid=213088400#gid=213088400" --day "3"
python3 extract_content_plan.py "program.tsv" --day "22.07" -o ~/Documents/ae_plaque_data/content_plan_day3
```

Парсер берет день из строк вида `ДЕНЬ 3 · 22.07`, время из первой колонки, а площадки строго из колонок `B/C/D`. Названия площадок читаются из строки заголовков таблицы, например `B = Амфитеатр`, `C = Урал 1 (синий)`, `D = Урал 2 (красный)`; подписи количества мест вроде `(200 мест)` в название не попадают. Контент в других колонках не превращается в `Колонка 2`, а попадает в warnings отчета. Темы выделяются из `Тема:` или из названия события перед блоком `Эксперт:`/`Гости:`/`Модератор:`. Для тем сессий в TSV пишется отдельное `ИМЯ_КОМПОЗИЦИИ`, например `Амфитеатр/Культурный суверенитет: какие смыслы держат страну`. При импорте тем старые композиции ищутся по тексту слоя `ТЕМА` и переименовываются без перезаписи внутренних слоев. Люди дедуплицируются в `content_plan_people.tsv`, повторные появления хранятся отдельно в `content_plan_session_people.tsv`. Плашки строятся по каждому появлению человека во времени, поэтому один спикер в двух сессиях даст две композиции с разным временем.

Если в программе встречается короткое имя с инициалами, например `Д.В. Аширов`, парсер дополнительно смотрит справочник `ФИО`/`Должность`:

```text
https://docs.google.com/spreadsheets/d/1J6nJHM4wXF66LJO7dDNT6QgrxlQ5VPb-3B-4o7Ff0js/edit?gid=1399617264#gid=1399617264
```

Совпадение ищется по инициалам и фамилии. Если найдено `Аширов Денис Валерьевич`, в TSV попадут полное ФИО и должность из справочника. Количество совпадений показывается в отчете как `Найдено в справочнике ФИО`. Если справочник недоступен, основной импорт не падает, но добавляет предупреждение. В терминале справочник можно заменить через `--people-ref-url` или отключить через `--no-people-ref`.

Целевая PostgreSQL-схема для будущего backend зафиксирована в:

```text
migrations/content_plan_postgres.sql
```

## Плашки Для Записи

Используйте `recording_plates_from_sheet.jsx`, если нужно собрать плашки записи из матричной таблицы:

```text
https://docs.google.com/spreadsheets/d/1J6nJHM4wXF66LJO7dDNT6QgrxlQ5VPb-3B-4o7Ff0js/edit?gid=1944136331#gid=1944136331
```

Скрипт читает диапазон `U1:AM25`:

- `U1:AM1` проверяется на маркер `ВИДЕО`;
- даты берутся из `V2:AM2`;
- время берется из `U3:U25`;
- имена людей берутся из пересечений даты и времени;
- служебные слова вроде `Обед`, `Перерыв`, `Завтрак` игнорируются;
- ФИО и должности обогащаются из справочника `gid=0` с колонками `ФИО` и `Должность`.

Названия колонок справочника можно задать прямо в окне: `Колонка ФИО`, `Колонка должности`, `Колонка фото`. Кнопка `Сохранить` записывает эти настройки в `~/Documents/ae_recording_plates_settings.json`.

После `Подготовить TSV` нажмите `Открыть плашки`. Откроется обычный генератор `person_plates_from_sheet.jsx`, но уже с TSV записи, форматом имен:

```text
20.07_10-00_Фамилия Имя
```

и папкой композиций:

```text
Запись
```

Если композиция с таким именем уже есть в папке `Запись`, повторно она не создается.

## Темы Сессий

Используйте `session_topics_from_sheet.jsx`, если нужно создать отдельную композицию на каждую тему сессии. Перед применением скрипт показывает план изменений: новые композиции, обновления существующих, строки без изменений и конфликты имен.

### Что Делает

- берет TSV/CSV/TXT файл, Google Sheet export URL или обычную Google Sheet edit-ссылку;
- в режиме `Программная таблица` достает `ТЕМА` из строки `Тема:`;
- в режиме `Программная таблица` берет `ОПИСАНИЕ` из текста перед файлом сценария, например `Главная встреча дня`;
- в режиме `Программная таблица` сканирует только колонки B, C и D;
- для B/C/D берет названия площадок из строки заголовков и пишет имя композиции вида `Площадка/Тема`, без подписи количества мест;
- считает ячейку темой только если в ней есть `Эксперт:` или `Эксперты:`;
- допускает пустое `ОПИСАНИЕ`;
- ищет главную композицию, по умолчанию `Главная`;
- дублирует ее только для новых уникальных строк;
- меняет Source Text в слоях `ТЕМА` и `ОПИСАНИЕ`;
- при повторном запуске обновляет только темы, где реально изменились `ТЕМА` или `ОПИСАНИЕ`;
- сохраняет служебный ключ в `Comment`, чтобы находить темы после ручного переименования композиции;
- сохраняет очищенный TSV для ручной правки;
- опционально добавляет созданные композиции в Render Queue.

### Простой TSV

Пример есть в `session_topics_example.tsv`:

```tsv
ТЕМА	ОПИСАНИЕ	ИМЯ_КОМПОЗИЦИИ
Название первой сессии	Описание первой сессии	Амфитеатр/Название первой сессии
Название второй сессии	Описание второй сессии	Урал 1 (синий)/Название второй сессии
```

### Запуск В AE

1. Откройте проект After Effects.
2. Подготовьте главную композицию `Главная`.
3. Внутри `Главная` подготовьте текстовые слои `ТЕМА` и `ОПИСАНИЕ`.
4. Запустите `File > Scripts > session_topics_from_sheet.jsx`.
5. Выберите режим `Файл TSV/CSV/TXT` или вставьте Google Sheet URL.
6. Проверьте имена главной композиции и текстовых слоев.
7. Нажмите `Создать композиции`.

### Режим Программной Таблицы

Если исходная программа лежит в большой таблице с ячейками вида `Главная встреча дня`, `Сценарий_...docx`, `Тема: ...`, оставьте включенным режим `Программная таблица`.

Скрипт дополнительно сохраняет редактируемый TSV:

```text
~/Documents/ae_plaque_data/session_topics_extracted.tsv
```

После первого прогона можно открыть этот TSV, удалить лишние строки или поправить `ОПИСАНИЕ`, затем выбрать в AE режим `Файл TSV/CSV/TXT` и указать отредактированный файл.

## Частые Проблемы

### `HTTP 401 Unauthorized`

Python не имеет доступа к Google Sheets. Откройте доступ к таблице для просмотра по ссылке, опубликуйте лист в CSV/TSV или скачайте таблицу как локальный CSV/TSV.

### AE не видит новые скрипты

Перезапустите After Effects после установки. Скрипты должны лежать в папке `Scripts`, а не `ScriptUI Panels`.

### Не найден текстовый слой

Проверьте имя слоя в AE и настройки во вкладке `Слои`. Для фото-плашек есть fallback по номеру слоя: `ИМЯ` N3, `ДОЛЖНОСТЬ` N4, фото N6.

### Файл рендера все еще с должностью

Выделите композиции в Project и запустите `shorten_render_queue_names.jsx`. Скрипт сам добавляет их в Render Queue и ставит короткое имя файла в `<папка проекта>/EXPORT`.

### В Render Queue вместо кириллицы `????????`

Обновите `shorten_render_queue_names.jsx` в папке AE Scripts через установщик и перезапустите After Effects. Актуальная версия создает файл через ASCII-заглушку в `EXPORT`, затем меняет имя через `File.changePath`, чтобы AE не терял русские буквы при присваивании `outputModule.file`.
